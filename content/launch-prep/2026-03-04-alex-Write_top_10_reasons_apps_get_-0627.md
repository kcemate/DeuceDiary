3. **Privacy Issues:** Apple is extremely strict about user data. Apps must be transparent about what data they collect, how they use it, and how they protect it.  Using tracking technologies without proper disclosure can lead to rejection.
4. **Payment Issues:** Problems with in-app purchases, subscriptions, or misleading pricing structures are frequent causes of rejection.
5. **Performance Issues:** Apps need to be stable, responsive, and not drain excessive battery life.  Crashing apps, slow loading times, and bugs are major red flags.
6. **Intellectual Property Violations:** Using copyrighted music, artwork, characters, or code without permission is a serious offense.
7. **Malware or Security Risks:** Apps must not contain malware, viruses, or other security vulnerabilities. Apple actively scans for these.
8. **Functionality Issues:** Apps need to operate as described and fulfill their core purpose. If the app consistently fails to do what it says it will, it will be rejected.
9. **App Store Guidelines Violations (Specific):**  This overlaps with #1, but includes issues like inappropriate advertising, using disallowed practices (e.g., using bot traffic), or violating specific guidelines related to certain categories (e.g., gambling apps).
