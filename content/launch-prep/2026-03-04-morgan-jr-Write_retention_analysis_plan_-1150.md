## Retention Analysis Plan

**Goal:** To understand user behavior following acquisition and identify key areas for improvement to increase user retention across key timeframes (D1, D7, and D30).

**Methodology:** This plan utilizes a multi-faceted approach combining quantitative data analysis with qualitative insights to provide a comprehensive retention analysis.

**I. Data Sources:**

* **Product Analytics:** (e.g., Mixpanel, Amplitude, Google Analytics) – Core user behavior data (event tracking, feature usage, session duration, conversion rates)
* **CRM/Customer Database:** User demographics, signup sources, initial onboarding information.
* **Support Tickets/Feedback Channels:** User pain points, common issues, and areas of frustration.
* **User Surveys:**  Gathering direct user feedback on satisfaction, engagement, and potential churn reasons (sent strategically – see timing below).


**II. Key Metrics & Definitions:**

* **DAU (Daily Active Users):**  Number of unique users active on a given day.
* **WAU (Weekly Active Users):** Number of unique users active in a given week.
* **MAU (Monthly Active Users):** Number of unique users active in a given month.
* **Retention Rate:** Percentage of users who return to the product after a specific period. (e.g., D1 Retention: Percentage of users who returned after 1 day).
* **Churn Rate:** Percentage of users who stopped using the product after a specific period. (Calculated as 1 - Retention Rate)
* **Cohort Analysis:**  Grouping users based on their acquisition date to track retention trends over time.
* **Time to Value (TTV):**  The time it takes for a new user to experience the core value proposition of the product.  A shorter TTV often correlates with higher retention.
