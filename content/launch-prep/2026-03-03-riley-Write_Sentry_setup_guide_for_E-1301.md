## Sentry Setup Guide for Expo React Native + Express

This guide outlines how to integrate Sentry into your Expo React Native project alongside your Express backend. It covers installation, configuration, and best practices for effective monitoring.

**Prerequisites:**

*   **Expo CLI:**  Make sure you have Expo CLI installed globally: `npm install -g expo-cli`
*   **React Native Project:** You have an existing Expo React Native project.
*   **Express Backend:** You have a running Express backend application.
*   **Sentry Account:**  You have a Sentry account and have created a project. (Get an API key from the Sentry dashboard).


**Step 1: Install Sentry Packages**

Install the necessary Sentry packages for both React Native and Express:

```bash
# For React Native
npx expo install @sentry/react-native

# For Express
npx install @sentry/node
```

**Step 2: Configure React Native Sentry**

1.  **Install `sentry-expo`:**  This package handles the Sentry SDK integration in Expo.

    ```bash
    npx expo install sentry-expo
    ```

2.  **Update `package.json`:**  Add the `sentry-expo` package as a dependency.

3.  **Initialize Sentry in your Expo App:**

    *   Navigate to the root of your React Native project.
    *   Run the following command:

        ```bash
        npx sentry-expo init
        ```

        This command will prompt you with a series of questions:

        *   **Sentry SDK Name:**  (Generally, you can leave this as the default or choose a descriptive name).
        *   **Project:** Select the Sentry project you created in the Sentry dashboard.
        *   **API Key:** Paste the Sentry API key you obtained from the Sentry dashboard.
        *   **Enable Redo:**  (Optional) Enable redo to capture re-thrown exceptions.  This helps you debug complex issues.
        *   **Enable Session Replay:** (Optional - Requires a paid Sentry plan)  This allows you to record user sessions and replay them for detailed debugging.
        *   **Enable Auto Track:** (Recommended)  Automatically track events like app start, stop, and user interaction for overall app health monitoring.

4. **Import and Initialize Sentry in your App:**  In your root app file (e.g., `App.js` or `App.tsx`), import and initialize Sentry:

   ```javascript
   import Sentry from 'sentry-expo';

   Sentry.init({
       dsn: 'YOUR_SENTRY_DSN', // Replace with your DSN from the init command
       // Enable session replay if you have a paid plan
       // enableSessionReplay: true,
   });
   ```

   *   **DSN:** The DSN (Data Source Name) is automatically generated by the `sentry-expo init` command and contains your project ID, etc.  You can also retrieve it from the Sentry dashboard.
