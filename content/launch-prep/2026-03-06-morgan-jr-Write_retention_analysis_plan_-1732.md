**Retention Analysis Plan**

**1. Goals & Objectives**

* **Overall Goal:** Understand why users are churning and identify opportunities to improve user engagement and retention.
* **Specific Objectives:**
    * Determine key drop-off points in the user journey.
    * Identify segments of users with high vs. low retention.
    * Measure the impact of specific onboarding or engagement initiatives.
    * Establish baseline retention rates and track progress over time.


**2. Data Requirements**

* **User Identification:** Unique user ID for each user.
* **Registration/Signup Date:** Date the user initially signed up.
* **Active Days:**  Number of days a user actively used the product/service (Define "active" – logging in, completing a key action, etc.). *This is the core metric.*
* **User Attributes (Segmentation):**  Demographics (age, location), acquisition source (e.g., paid ad, organic search, referral), plan type, initial usage behavior, etc.  The more segments you can create, the richer the insights.
* **In-App Events:** Track key user actions within the product (e.g., completed tutorial, added item to cart, made a purchase, created a project).
* **Churn Definition:**  Clearly define what constitutes a "churned" user. (e.g., no activity for 30 days, account cancellation).

**3. Benchmarks & Metrics**

**4. Methodology & Analysis**

* **Cohort Analysis:** This is *crucial*. Group users based on their signup date (cohorts). Track the retention rate of each cohort over time. This reveals trends over time.
* **Funnel Analysis:** Examine the user flow from signup to key actions. Identify where users are dropping off in the funnel.
* **Segment Analysis:**  Compare retention rates across different user segments (e.g., by acquisition source, plan type, demographics).
* **Event Tracking Analysis:** Correlate retention with specific in-app events. Are users who complete the tutorial more likely to stay?
