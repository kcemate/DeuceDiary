**II.  Paywall Strategy - When & How**

* **Trigger Points for Paywall Activation:**
    * **Behavioral Thresholds (Most Important):**
        * **Time Spent in Key Features:** Track time spent within the core features that provide the most value.  After a certain duration (e.g., 30-60 minutes) in these features, introduce a paywall prompt.
        * **Number of Key Actions Taken:** Track actions that represent significant progress or value (e.g., creating a project, analyzing a report, completing a task).  After a certain number (e.g., 3-5), prompt.
        * **Feature Depth:**  If a user spends a long time in a specific area of your product, it suggests they're using it for a serious purpose.
        * **Engagement Metrics:**  Daily/Weekly Active Users, Session Length – high engagement signals higher potential value.
    * **Time-Based Introduction (Initial):**  After a short free trial (see below), subtly introduce the concept of premium features. Don't push it aggressively.
    * **Segmented Offers:**  Based on user profile (industry, job title, etc.) - offer targeted features/access relevant to their needs.
* **Paywall Types:**
    * **Gradual Paywall:** Initially, limit access to key features.  As users engage further, unlock more.
    * **Feature-Based Paywall:**  Offer a tiered model – Basic (free), Pro (mid-tier), Enterprise (high-tier) – each with a different feature set.
    * **Usage-Based Paywall:** (Potentially for SaaS) – Charge based on usage (e.g., number of users, API calls, data volume).

**III. Free Trial Length - Experimentation is Key**

* **Start with a Short Trial:** 7-14 days is a good starting point. This forces users to quickly experience the value.
* **A/B Test Trial Lengths:**  Crucially, A/B test different trial lengths (7, 14, 30 days) and conversion rates.  Longer trials can be effective if you’re providing truly transformative value.
* **Consider Value-Based Trials:** Instead of a fixed duration, a trial could unlock full access to key features for a limited time *relative to the user's expected use case*. For example, a 30-day trial that unlocks the full suite of analytics tools if a user indicates they’re analyzing large datasets.
