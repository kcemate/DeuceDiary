**Phase 1: Core Team & Close Friends (1 Week)**

* **Feedback Focus:**  Absolutely *everything*. This is a ‘vomit test’ – prioritize identifying crashes, bugs, major UI glitches, and any confusing elements. Usability of core diary creation, entry methods, and basic organization.

**Phase 2: “Power Users” – Focused User Group (2 Weeks)**
* **Invited:** 10-15 users recruited through existing channels (Discord server, email list, social media posts emphasizing early access). * **Feedback Focus:** Deep dive into the diary experience. Request feedback on:

    * **Import/Export (if applicable):** Test this thoroughly.
    * **Dark Mode:** Feedback on appearance.
* **Tools:** TestFlight, UserVoice (for feedback collection), specific questions within the app (e.g., “Rate your experience with…”).
* **Target:** 10-15 users

**Phase 3: Broad Beta – Expanding the User Base (3-4 Weeks)**
* **Invited:**  Increase the invite pool to 50-100 users.  Continue to use recruitment channels.  Consider running small, targeted ads on social media.
* **Feedback Focus:** Monitor trends from earlier phases.  Focus on high-impact areas identified during Phase 2.  Start to test less-core features if they’ve been built.  Pay close attention to user retention – are people actively using the app?
* **Tools:** TestFlight, NPS surveys, In-App Feedback (if integrated), Google Forms for more structured feedback.  Prioritize bug reports with clear steps to reproduce.
* **Target:** 50-100 users
