## Sentry Setup Guide for Expo React Native + Express

This guide outlines the steps to integrate Sentry into your Expo React Native app and your Express backend, providing comprehensive error tracking and diagnostics.

**Prerequisites:**

*   **Sentry Account:** You'll need a Sentry account. Sign up at [https://sentry.io/](https://sentry.io/).
*   **Expo CLI:** Ensure you have Expo CLI installed globally: `npm install -g expo-cli`
*   **Node.js & npm/yarn:**  Make sure you have Node.js and a package manager (npm or yarn) installed.
*   **Expo Project:** You have an existing Expo React Native project.

**1. Create a Sentry Project in Sentry:**

*   Log in to your Sentry account.
*   Click "Create project" (or "New project" if you're starting from scratch).
*   Choose the "Application" option.
*   Give your project a name (e.g., "MyExpoApp").
*   Select your SDK (usually "Node.js").
*   Choose your environment (e.g., "Production").
*   Click "Create project."
*   **Important:** Note the **Sentry SDK key** (also known as the **Publish Key**) provided by Sentry. You'll need this to configure your app.

**2. Install Sentry Packages:**

*   Navigate to your Expo project's root directory in your terminal.
*   Install the Sentry SDK for React Native:
    ```bash
    npm install @sentry/react-native
    # or with yarn
    yarn add @sentry/react-native
    ```
*   If you have an Express backend, install the Sentry Node.js SDK:
    ```bash
    npm install @sentry/node
    # or with yarn
    yarn add @sentry/node
    ```

**3. Configure React Native App (Expo):**

*   **`sentry/init/native.js` (or equivalent):** Create this file in your project if it doesn't already exist. This file is crucial for initializing Sentry on the Expo side.  Place it in the root of your project.

    ```javascript
    import * as Sentry from '@sentry/react-native';

    // Replace with your Sentry SDK key
    const SENTRY_SDK_KEY = 'YOUR_SENTRY_SDK_KEY';

    Sentry.init({
      dsn: SENTRY_SDK_KEY,
      // Set environment (e.g., 'production')
      environment: 'development',
      //  Automatic context linking (optional - for better debugging)
      //  this will link snippets from the mobile app to the backend.
      tracesSampleRate: 0.2,  // Adjust based on your needs
    });
    ```

    *   **Replace `YOUR_SENTRY_SDK_KEY`** with the actual SDK key you obtained from your Sentry project.
    *   **`environment`:**  Setting the environment helps you filter and categorize errors in the Sentry dashboard. Use 'development', 'staging', or 'production'.
    *   **`tracesSampleRate`:**  Determines the percentage of traces to sample. Higher values provide more data but increase processing overhead.  Start with a low value and increase as needed.
    *   **Automatic Context Linking:**  Enabling this can be immensely helpful for debugging issues that span both the mobile and backend.  It automatically adds context from the mobile app (e.g., user ID, screen name) to Sentry traces generated by your backend.
